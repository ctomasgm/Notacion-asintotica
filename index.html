Notación θ grande (Big-θ)
Cuando decimos que un tiempo de ejecución particular es de Θ(n), estamos diciendo que una vez que n n nn sea suficientemente grande, el tiempo de ejecución será por lo menos k1⋅n y a lo más k2⋅n para algunas constantes k1 y k2.
Notación O grande (Big-O)
Usamos la notación Θ grande para acotar de manera asintótica el crecimiento de un tiempo de ejecución dentro del rango de factores constantes por arriba y por abajo. A veces queremos acotar solo por arriba. Si un tiempo de ejecución es O(f(n)), entonces para n suficientemente grande, el tiempo de ejecución es a lo más k⋅f(n) para alguna constante k. Decimos que el tiempo de ejecución es "O grande de f(n)" o simplemente "O de f(n)". Usamos la notación O grande para cotas superiores asintóticas, ya que acota el crecimiento del tiempo de ejecución por arriba para entradas suficientemente grandes.
Notación Omega grande (Big-Ω)
A veces, queremos decir que un algoritmo toma por lo menos una cierta cantidad de tiempo, sin dar una cota superior. Utilizamos la notación Ω grande; esa es la letra griega "omega". Decimos que el tiempo de ejecución es "Ω grande de f(n)". Usamos la notación Ω grande para límites asintóticos inferiores, ya que acota el crecimiento del tiempo de ejecución por abajo para entradas de tamaños suficientemente grandes.
